## 제1조 (목적)

본 계약은 갑이 을에게 의뢰하는 소프트웨어 개발에 관하여 양 당사자 간의 권리와 의무를 명확히 함을 목적으로 한다.

## 제2조 (개발 대상)

1. **프로젝트명**: 푸른나무재단 강사 배정 관리 시스템
2. **개발 내용**: 별첨 "요구사항 명세서"에 명시된 기능
3. **개발 범위**: 본 계약서 및 요구사항 명세서에 명시된 범위로 한정한다.

## 제4조 (개발 비용 및 지급)

1. **총 개발 비용**: 금 ____________원 (₩____________)
2. **지급 방법**:

   - 중도금: 개발 __% 완료 시 총액의 __% (₩________)

## 제5조 (개발 범위 및 변경)

1. 개발 범위는 본 계약 체결 시 합의된 "요구사항 명세서"에 한정한다.
2. **명세서에 명시되지 않은 기능은 개발 범위에 포함되지 않는다.**
3. 갑이 추가 기능 또는 변경을 요청할 경우:
   - 을은 추가 비용 및 일정을 산정하여 갑에게 통보한다.
   - 갑이 동의할 경우 별도의 추가 계약을 체결한다.
4. 개발 진행 중 갑의 요청으로 인한 일정 지연은 을의 책임이 아니다.

---

## 제6조 (검수)

1. 을은 개발 완료 후 갑에게 검수를 요청한다.
2. **검수 기준**: 요구사항 명세서에 명시된 기능이 정상 동작하는지 여부
3. **검수 기간**: 검수 요청일로부터 __일 (영업일 기준)
4. 갑은 검수 기간 내에 검수 결과를 을에게 서면 통보해야 한다.
5. **검수 기간 내 별도 통보가 없을 경우, 검수가 완료된 것으로 간주한다.**
6. 검수 불합격 시 갑은 구체적인 사유를 명시하여 서면 통보해야 하며, 해당 사유는 요구사항 명세서에 근거해야 한다.

---

## 제7조 (하자보수)

1. **하자보수 기간**: 검수 완료일로부터 __개월
2. **하자보수 범위**: 요구사항 명세서에 명시된 기능의 오작동 수정
3. **하자보수 범위에 포함되지 않는 사항**:
   - 갑 또는 제3자의 임의 수정으로 인한 오류
   - 운영 환경 변경(서버, OS, 브라우저 등)으로 인한 오류
   - 제3자 서비스(API, 라이브러리 등)의 변경 또는 중단으로 인한 오류
   - 요구사항 명세서에 명시되지 않은 기능
   - 신규 기능 추가 또는 기존 기능 변경
4. 하자보수 기간 종료 후 발생하는 수정 요청은 별도 유지보수 계약에 따른다.

---

## 제8조 (면책)

1. 을은 다음 사항에 대해 책임을 지지 않는다:
   - 요구사항 명세서에 명시되지 않은 기능 또는 동작
   - 갑이 명확하게 전달하지 않은 요구사항
   - 갑 또는 제3자의 부주의, 오용, 무단 수정으로 인한 손해
   - 제3자 서비스(카카오 API, 호스팅 서비스 등)의 장애, 변경, 중단
   - 갑의 운영 환경(서버, 네트워크, 보안 설정 등)으로 인한 문제
   - 법령 변경으로 인한 서비스 수정 필요
2. 을의 최대 손해배상 책임은 갑이 을에게 지급한 개발 비용을 초과하지 않는다.

---

## 제9조 (민감정보 및 보안)

1. **민감정보의 정의**: 본 시스템에서 다루는 군부대 정보(부대명, 위치, 교육 일정 등)는 민감정보로 분류한다.
2. **법령 준수 책임**:
   - 군사기밀보호법, 국가정보원법 등 관련 법령의 준수 여부 확인 및 이행 책임은 **갑**에게 있다.
   - 을은 갑이 제공한 데이터가 법적으로 시스템에 저장/처리 가능한 것으로 신뢰한다.
   - 갑은 본 시스템에 입력되는 군부대 정보가 관련 법령에 저촉되지 않음을 보증한다.
3. **데이터 제공 책임**:
   - 갑이 을에게 제공하는 군부대 정보의 정확성과 적법성은 갑이 책임진다.
   - 을은 갑이 제공한 정보를 그대로 시스템에 반영하며, 해당 정보의 내용에 대해 책임지지 않는다.
4. **보안 조치**:
   - 을은 개발 시 합리적인 수준의 보안 조치(비밀번호 암호화, SQL 인젝션 방지 등)를 적용한다.
   - 단, 운영 환경의 보안 설정(서버 방화벽, SSL 인증서, 접근 제어 등)은 갑의 책임이다.
5. **보안 사고 면책**:
   - 을은 다음으로 인한 보안 사고에 대해 책임지지 않는다:
     - 갑의 보안 설정 미비 또는 관리 소홀
     - 갑 또는 갑의 사용자의 계정 정보 유출
     - 제3자의 해킹, DDoS 공격 등 외부 공격
     - 갑이 을의 권고를 무시한 경우
   - 단, 을의 명백한 개발 과실(알려진 취약점 미조치 등)로 인한 사고는 예외로 한다.
6. **운영 책임**:
   - 검수 완료 후 시스템의 운영, 데이터 관리, 백업, 보안 유지는 갑의 책임이다.
   - 을은 운영 중 발생하는 데이터 유실, 유출, 훼손에 대해 책임지지 않는다.

---

## 제10조 (지식재산권)

1. 본 개발 결과물의 저작권은 잔금 지급 완료 시 갑에게 귀속된다.
2. 단, 을이 개발에 사용한 기존 라이브러리, 프레임워크, 범용 코드의 권리는 을에게 유보된다.
3. 을은 본 프로젝트를 포트폴리오 목적으로 사용할 수 있다. (선택사항 - 민감정보 특성상 삭제 권장)

   - 개발 진행률에 따라 추가 정산한다.
3. 을의 귀책사유로 계약 해제 시:
   - 기 지급된 금액에서 개발 진행률에 해당하는 금액을 공제 후 반환한다.
